List of concepts for the PolarSys Maturity Assessment quality model

Concept name

Control flow complexity (CPX_CF)

Description: The computational complexity of the control flow structure in the code functions.

Concept is active.

Composed of VG.

Concept name

Data flow complexity (CPX_DF)

Description: The complexity of the data flow structure in the code functions.

Concept is active.

Composed of DOPD, DOPT, TOPD, TOPT.

Concept name

OO complexity (CPX_OO)

Description: The complexity of the object-oriented architecture and structure. Composed of MDIT (maximum depth of inheritance tree).

Concept is active.

Composed of MDIT.

Concept name

Code documentation (CODE_DOC)

Description: The amount of documentation in the code: comment rate, javadoc, etc.

Concept is active.

Composed of COMR.

Concept name

Code size (CODE_SIZE)

Description: The overall size of the code.

Concept is active.

Composed of SLOC.

Concept name

Access points (ALL_DATA)

Description: The amount of attributes and methods on the class, whatever their visibility is.

Concept is active.

Composed of ATTRS, METHS.

Concept name

Public access points (PUBLIC_DATA)

Description: Total number of public classes, interfaces, methods, constructors, annotations and attributes.

Concept is active.

Composed of ATTRS_PUBLIC, METHS_PUBLIC.

Concept name

IP Log (IP_LOG)

Description: Is the intellectual property of contributions checked and enforced?

Concept is active.

Composed of IP_LOG_COV.

Concept name

SCM Documentation (SCM_DOCS)

Description: Is there some documentation about the SCM process, rules and conventions?

Concept is active.

Concept name

ITS information access (PUB_ITS_INFO)

Description: Measures the availability of information regarding ITS usage, conventions, access.

Concept is active.

Composed of ITS_ISSUES_OPEN_1M, ITS_ISSUES_OPENERS_1M.

Concept name

ITS usage (ITS_USAGE)

Description: How much the ITS system was used during last month.

Concept is active.

Composed of ITS_UPDATES_1M, ITS_AUTH_1M.

Concept name

SCM information access (PUB_SCM_INFO)

Description: Measures the availability of information regarding SCM usage, conventions, access.

Concept is active.

Concept name

SCM usage (SCM_USAGE)

Description: How much the SCM system was used during last month.

Concept is active.

Composed of SCM_COMMITS_1M, SCM_COMMITTERS_1M.

Concept name

SCM Stability Index (SCM_STABILITY_IDX)

Description: How much the files were stable according to the SCM system.

Concept is active.

Composed of SCM_COMMITS_1M, FILES.

Concept name

Code analysability index (CODE_ANA_IDX)

Description: A weighted index of code characteristics that impact analysability.

Concept is active.

Concept name

Non conformities index for analysability (RULES_ANA_IDX)

Description: Number of non conformities to analysability rules found in the artefact, divided by the number of files.

Concept is active.

Composed of NCC_ANA_IDX.

Concept name

Analysability practices acquired (RULES_ANA_OK)

Description: Ratio of conform (non-violated) analysability practices on the number of checked analysability practices.

Concept is active.

Composed of ROKR_ANA.

Concept name

Code changeability index (CODE_CHA_IDX)

Description: A weighted index of code characteristics that impact changeability.

Concept is active.

Concept name

Non conformities index for changeability (RULES_CHA_IDX)

Description: Number of non conformities to changeability rules found in the artefact, divided by the number of files.

Concept is active.

Composed of NCC_CHA_IDX.

Concept name

Changeability practices acquired (RULES_CHA_OK)

Description: Ratio of conform changeability practices on the number of checked analysability practices.

Concept is active.

Composed of ROKR_CHA.
